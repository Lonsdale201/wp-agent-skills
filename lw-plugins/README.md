# lw-plugins

Skills covering the **LW Plugins family** (LW LMS, LW Site Manager). Use these when extending or building on top of these plugins.

## Skills

| Skill | Purpose |
|---|---|
| `lw-lms-backend-extend` | Backend extension contract for `lwplugins/lw-lms` (BETA — README explicitly says "not recommended for production use"), verified for v1.3.0. Headless LMS — courses / lessons / sections / progress / access control. Six verified actions including the canonical enrollment event `lw_lms_after_grant` (5 args) and `lw_lms_after_revoke` (3 args), plus three filters — `lw_lms_pre_grant` (abort grant), `lw_lms_has_course_access`, `lw_lms_has_lesson_access`. Public API split read vs write — `AccessRepository` / `ProgressRepository` for writes, `AccessQueries` / `ProgressQueries` for reads. `ProgressRepository::mark_course_completed()` is the canonical force-complete entry. 11 custom capabilities, three DB tables, optional Site Manager integration. Routes abilities, frontend, and migration topics to sibling skills. |
| `lw-lms-rest-frontend` | Build a custom frontend against `lwplugins/lw-lms`'s headless `/wp-json/lms/v1` REST API (BETA, verified for v1.3.0). The plugin is intentionally HEADLESS — no shipped templates, shortcodes, or blocks. Covers the public list/single course endpoints, auth+access lessons / progress GET+POST / per-course progress / download endpoints. Single-course response includes `content` ONLY when `access.has_access === true`. Paid-course `access` carries `products`, `subscriptions`, AND `subscription_variations` (variation-level WC subscription upsells, since 1.2.15). |
| `lw-lms-abilities` | Consumer / reviewer reference for the built-in `lw-lms/*` Abilities API surface registered by lw-lms v1.3.0 — `list-courses`, `get-course`, `get-progress`, `set-progress`, `get-options`. Dual registration model — Site Manager bridge (`lw_site_manager_register_abilities`) when present, standalone WP 6.9+ Abilities API fallback (`wp_abilities_api_init` priority 20) otherwise; `did_action()` guards prevent duplicate registration. Calling pattern via `/wp-json/wp-abilities/v1/abilities/lw-lms/.../run` with `input`-wrapped JSON body. Output schemas added per ability since v1.2.16. |
| `lw-lms-learndash-migration` | Plan, run, or review the one-time WP-CLI LearnDash → LW LMS migration (`wp lw-lms migrate-learndash --dry-run --verbose`) in lw-lms v1.3.0. Maps `sfwd-courses` / `sfwd-lessons` into `course` / `lesson` posts; writes `_lw_lms_migrated_to` reverse mappings on source posts; builds one generated section per course from `ld_course_steps`; extracts LearnDash / Elementor / embed video URLs into `_lw_lms_video`. Rerunnable but not a sync — does NOT migrate quizzes, certificates, groups, assignments, user progress, enrollments, or drip rules. |
| `lw-site-manager-overview` | Reference for `lwplugins/lw-site-manager` — a WP 6.9+ Abilities-API-native exposure layer that registers 120+ machine-callable abilities under `site-manager/*` for AI agents (Claude, ChatGPT, MCP clients) to discover and invoke. Calling pattern via `/wp-json/wp-abilities/v1/abilities/{namespace}/{ability}/run` with Application Password Basic auth. **Not** MainWP — different surface and security model (per-ability cap-checks). |
| `lw-site-manager-extend-abilities` | Add custom abilities to LW Site Manager via two action hooks (`lw_site_manager_register_categories`, `lw_site_manager_register_abilities` — second receives the central `PermissionManager` instance). Critical pattern — extend `AbstractAbilitiesRegistrar` to inherit the meta builders (`readOnlyMeta` / `writeMeta` / `destructiveMeta`) and schema builders (`paginationSchema` / `orderingSchema` / `idSchema` / `listOutputSchema` / etc.). |

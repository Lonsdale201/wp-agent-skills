# lw-plugins

Skills covering the **LW Plugins family** (LW LMS, LW Site Manager). Use these when extending or building on top of these plugins.

## Skills

| Skill | Purpose |
|---|---|
| `lw-lms-backend-extend` | Backend extension contract for `lwplugins/lw-lms` (BETA — README explicitly says "not recommended for production use"), verified for v1.5.1. Headless LMS — courses / lessons / sections / progress / access control. Verified actions including the canonical enrollment event `lw_lms_after_grant` (5 args), `lw_lms_after_revoke` (3 args), and `lw_lms_attachment_downloaded`, plus filters — `lw_lms_pre_grant` (abort grant), `lw_lms_has_course_access`, `lw_lms_has_lesson_access`, and the v1.4.0 `lw_lms_settings_tabs` (companion settings tabs sharing `options.php` via `SettingsPage::get_settings_group()`). Public API split read vs write — `AccessRepository` / `ProgressRepository` for writes, `AccessQueries` / `ProgressQueries` for reads. `ProgressRepository::mark_course_completed()` is the canonical force-complete entry. v1.5.0 adds read-time WooCommerce Memberships access via `_lw_lms_membership_plan_ids` / `MembershipChecker` (no access row written). 11 custom capabilities, three DB tables, optional Site Manager integration. Routes abilities, frontend, migration, and WP-CLI-operations topics to sibling skills. |
| `lw-lms-wp-cli-operations` | Operational LW LMS WP-CLI commands added in v1.4.0, verified against v1.5.1 — `wp lw-lms course create\|list\|delete\|set-section`, `lesson create\|list\|assign`, `enroll`, `revoke`, `force-complete`. Covers `CliResolver` argument resolution by ID / slug / login / email, the `AccessRepository::grant()` / `revoke()` and `ProgressRepository::mark_course_completed()` side effects each command triggers, enrollment / progress hook firing, and the CLI footguns (`enroll` always passes `source_id = null` so `--source` alone doesn't create distinct rows; `set-section` and `lesson assign` are separate; `force-complete` writes progress not access; no dry-run). **Not** the LearnDash migration command — that stays in `lw-lms-learndash-migration`. |
| `lw-lms-rest-frontend` | Build a custom frontend against `lwplugins/lw-lms`'s headless `/wp-json/lms/v1` REST API (BETA, verified for v1.5.1). The plugin is intentionally HEADLESS — no shipped templates, shortcodes, or blocks. Covers the public list/single course endpoints, auth+access lessons / progress GET+POST / per-course progress / download endpoints. Since v1.4.0 single-course `content` is PUBLIC marketing/about content (no longer gated behind `access.has_access`); only lesson content stays access-gated. Denied paid-course `access` carries `products`, `subscriptions`, `subscription_variations` (variation-level WC subscription upsells, since 1.2.15), AND `memberships` (linked WooCommerce Memberships plans, since v1.5.0). |
| `lw-lms-abilities` | Consumer / reviewer reference for the built-in `lw-lms/*` Abilities API surface registered by lw-lms v1.5.1 — `list-courses`, `get-course`, `get-progress`, `set-progress`, `get-options` (the v1.4.0 WP-CLI/settings and v1.5.0 Memberships work added no new abilities). Dual registration model — Site Manager bridge (`lw_site_manager_register_abilities`) when present, standalone WP 6.9+ Abilities API fallback (`wp_abilities_api_init` priority 20) otherwise; `did_action()` guards prevent duplicate registration. Calling pattern via `/wp-json/wp-abilities/v1/abilities/lw-lms/.../run` with `input`-wrapped JSON body. Output schemas added per ability since v1.2.16. |
| `lw-lms-learndash-migration` | Plan, run, or review the one-time WP-CLI LearnDash → LW LMS migration (`wp lw-lms migrate-learndash --dry-run --verbose`) in lw-lms v1.5.1 (the v1.4.0 WP-CLI release added operational commands but did not change the `migrate-learndash` flow — see `lw-lms-wp-cli-operations` for those). Maps `sfwd-courses` / `sfwd-lessons` into `course` / `lesson` posts; writes `_lw_lms_migrated_to` reverse mappings on source posts; builds one generated section per course from `ld_course_steps`; extracts LearnDash / Elementor / embed video URLs into `_lw_lms_video`. Rerunnable but not a sync — does NOT migrate quizzes, certificates, groups, assignments, user progress, enrollments, or drip rules. |
| `lw-site-manager-overview` | Reference for `lwplugins/lw-site-manager` — a WP 6.9+ Abilities-API-native exposure layer that registers 120+ machine-callable abilities under `site-manager/*` for AI agents (Claude, ChatGPT, MCP clients) to discover and invoke. Calling pattern via `/wp-json/wp-abilities/v1/abilities/{namespace}/{ability}/run` with Application Password Basic auth. **Not** MainWP — different surface and security model (per-ability cap-checks). |
| `lw-site-manager-extend-abilities` | Add custom abilities to LW Site Manager via two action hooks (`lw_site_manager_register_categories`, `lw_site_manager_register_abilities` — second receives the central `PermissionManager` instance). Critical pattern — extend `AbstractAbilitiesRegistrar` to inherit the meta builders (`readOnlyMeta` / `writeMeta` / `destructiveMeta`) and schema builders (`paginationSchema` / `orderingSchema` / `idSchema` / `listOutputSchema` / etc.). |

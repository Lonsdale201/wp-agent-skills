# elementor

Developer skills for building **[Elementor](https://elementor.com)** addon/companion plugins — registering Dynamic Tags, wiring their controls and fallback behavior, picking from large datasets without freezing the editor, and auditing for deprecated Elementor APIs.

A key thread runs through the domain: the **Dynamic Tags infrastructure** (base classes, the registration manager, hooks, categories/groups) ships in **free** Elementor under `Elementor\Core\DynamicTags\*`, while the editor's tag picker and the AJAX query control are **Pro** — so the skills consistently extend the free base classes, feature-detect the Pro-only pieces, and degrade gracefully. Every claim is grounded in source line references against the tested version, and the deprecations skill is built so the list can be regenerated from source for whatever version you actually have installed.

## Skills

| Skill | Purpose |
|---|---|
| `elementor-dynamic-tag-register` | Register a custom Dynamic Tag from a companion plugin: hook the modern `elementor/dynamic_tags/register` action and `$manager->register( new MyTag() )` (the legacy `register_tags` + `register_tag( $class )` pair is deprecated since 3.5.0), the four required methods (`get_name` / `get_title` / `get_group` / `get_categories`), `register_group()`, bootstrap timing on `elementor/loaded`, extending the **free** `Tag` / `Data_Tag` bases for portability, and the Pro-feature reality (the API is free, the editor picker + AJAX control are Pro — feature-detect and degrade). |
| `elementor-dynamic-tag-fields` | Build the body of a Dynamic Tag: `Tag` (echoes via `render()`, content type `ui`) vs `Data_Tag` (returns via `get_value()`, content type `plain`); declaring the value kind via `get_categories()` with the `Module::*_CATEGORY` constants; adding settings in `register_controls()` (not the deprecated `_register_controls()`) with `Controls_Manager` types; and the fallback asymmetry that bites — a `Tag` gets Before / After / Fallback automatically, a `Data_Tag` gets none and must register and consult its own fallback. Bundled `reference.md` has the full control-type catalog and complete worked `Tag` / `Data_Tag` examples. |
| `elementor-dynamic-tag-ajax-select` | Let a control (in a tag or widget) pick one item from a large dataset (products, posts, terms, users) without freezing the editor: why a preloaded `SELECT2` hangs the panel at scale, Elementor Pro's AJAX query control (`QueryControlModule::QUERY_CONTROL_ID` + the `autocomplete` config with `QUERY_OBJECT_*`), why it never runs until the user types, using it inside a Dynamic Tag, and the required graceful degradation to a manual ID `TEXT` field via `class_exists( QueryControlModule::class )` when Pro is absent. |
| `elementor-deprecations` | Audit addon code for deprecated Elementor APIs and deprecate your own code correctly: the central `Deprecation` class (3.1+) and its six `deprecated_*` / `do_deprecated_action` / `apply_deprecated_filter` methods (each carrying name + version + replacement, so the list is greppable from source), the underscore→no-underscore method renames (`_register_controls` → `register_controls`, etc.), the 3.5.0 registration-hook renames (`*_registered` / `register_tags` / `categories/init` → `*/register`), and the debugging gotcha that Elementor's PHP notices need `WP_DEBUG` **and** `ELEMENTOR_DEBUG` within a 4-major window. Bundled `reference.md` has the extraction recipe and a per-version snapshot. |

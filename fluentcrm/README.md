# fluentcrm

Skills for extending the **[FluentCRM](https://fluentcrm.com)** automation / CRM plugin and its Pro add-on **FluentCampaign**. Use these when building a companion plugin that registers custom funnel triggers, actions, benchmarks, smart codes, dynamic segments, or `rest_selector` option lists — or that reads/writes FluentCRM contacts, companies, lists, tags, event-tracking activity, automation state, and email sequences through the public PHP API and ORM models.

The Free plugin (`fluent-crm`) ships the funnel chassis (BaseTrigger / BaseAction / BaseBenchMark, FunnelProcessor, FunnelHelper, the contact lifecycle hooks, the `fluentcrm_ajax_options_*` filter family). The Pro add-on (`fluent-crm-pro` / `FluentCampaign`) layers integrations and advanced action / benchmark types on top — see `fluentcrm-overview` for the split.

## Skills

| Skill | Purpose |
|---|---|
| `fluentcrm-overview` | Orient skill — Free vs Pro split, plugin paths and constants, bootstrap order (`fluentcrm_loaded` → `fluentcrm_addons_loaded` → `fluent_crm/after_init`), the model layer (Subscriber, Funnel, FunnelSequence, FunnelSubscriber, FunnelMetric), the global helpers (`FluentCrmApi`, `fluentCrmDb`, `FunnelHelper`), the contact lifecycle hooks (`fluent_crm/contact_created`, `_updated`, `_email_changed`, `_custom_data_updated`), the smart-code extension filter (`fluent_crm/extended_smart_codes`), and a decision matrix for picking the right extension contract. Read this first. |
| `fluentcrm-funnel-trigger` | Extend `BaseTrigger` to start a FluentCRM automation when something happens (course enrolled, SaaS webhook arrived, custom CPT published). Covers the four abstract methods, the auto-injected `__force_run_actions` field, the canonical `isProcessable` / `run_multiple` / `ifAlreadyInFunnel` guard, and the `source_trigger_name` / `source_ref_id` metadata. Critical lifecycle — register on `fluentcrm_loaded` priority below 10, NOT on `fluent_crm/after_init`; multi-arg hooks need an explicit `fluentcrm_funnel_arg_num_{name}` filter or extra args silently drop. |
| `fluentcrm-funnel-action` | Extend `BaseAction` to add a sequence step that runs per-contact when an automation reaches it. Three abstract methods, the per-step `handle($subscriber, $sequence, $funnelSubscriberId, $funnelMetric)` signature, and the canonical skip / failure semantics — `handle` ONLY overrides status on early-return, because `FunnelProcessor::processSequence` already marks the sequence `'complete'` BEFORE dispatch. Status string canon is `'complete'` (NOT `'completed'`). Same lifecycle rule as triggers. |
| `fluentcrm-funnel-benchmark` | Extend `BaseBenchMark` to add a goal / wait point that pauses sequence execution until a matching event occurs (tag applied, list joined, course completed, custom event). Three abstract methods, Optional vs Essential semantics, the `can_enter` direct-entry toggle, the `assertCurrentGoalState` filter, and `FunnelProcessor::startFunnelFromSequencePoint` as the canonical resume entry — NOT `startFunnelSequence` (that starts a new run). Same lifecycle and arg-num timing as triggers. |
| `fluentcrm-rest-options` | Register a custom AJAX option list for FluentCRM trigger / action / benchmark editor pickers. Pairs `'type' => 'rest_selector', 'option_key' => '<key>'` in a settings field with `add_filter('fluentcrm_ajax_options_<key>', $cb, 10, 3)` server-side. Filter signature `($options, $search, $includedIds)` returns `[{id, title}]`. Critical — pre-selected IDs must always be returned regardless of `$search` or the editor renders saved values as raw IDs instead of human labels. |
| `fluentcrm-contact-models` | Read and write FluentCRM contacts through the public API and ORM (core, no Pro). `FluentCrmApi('contacts')->createOrUpdate()` (delegates to `Subscriber::updateOrCreate()` — syncs lists/tags/custom fields, links a WP user, fires lifecycle hooks) instead of raw `$wpdb` or bare `Subscriber::create()`; contact lookup (`getContact` / `getContactByUserRef` / `getCurrentContact`); the status model (`subscribed` / `pending` / `transactional` / `unsubscribed` / `bounced` / `complained` / `spammed`) and the no-downgrade / strict-status write rules; `custom_values` and the `syncCustomFieldValues` delete flag; list/tag `importBulk` + `attachLists` / `attachTags` / `detach*` and the precise added/removed hooks; and `ContactsQuery` segment reads with allowlisted sort. Tested FluentCRM 3.1.8. |
| `fluentcrm-companies-model` | Work with FluentCRM 3.x Companies / B2B account records from a companion plugin — guard the experimental `company_module` flag (`Helper::isCompanyEnabled()`), `FluentCrmApi('companies')->createOrUpdate()` (lookup by id or exact name; `owner_id` is a Subscriber ID, not a WP user ID; `custom_values` via `CustomCompanyField`), the `Company` / `fc_companies` model, primary `company_id` vs the many-to-many pivot (`Company::subscribers()` / `Subscriber::companies()`), `attachContactsByIds` / `detachContactsByIds` with owner/primary reassignment on detach, the company hooks (`fluent_crm/company_created` / `_updated` / `_deleted`, legacy `fluentcrm_contact_added_to_companies`), and company-aware contact filters/segments. Corrects the `getCompany()` name-vs-email lookup trap. Tested FluentCRM 3.1.8. |
| `fluentcrm-event-tracking` | Track and consume FluentCRM 3.x contact events from a companion plugin — guard the experimental `event_tracking` flag (`Helper::isExperimentalEnabled('event_tracking')`; `track()` returns `WP_Error('not_enabled')` when off), `FluentCrmApi('event_tracker')->track()` with its subscriber-resolution order and required `event_key` / `title`, repeatable (counter increment) vs append-only rows, the `fluent_crm/track_event_activity` action bridge, the `fluent_crm/event_tracked` hook, the `event_tracking_*` advanced contact filters and `event_tracking_keys` option source, and FluentCampaign Pro's Tracking Event Recorded trigger / Add Event Tracking action. Tested FluentCRM 3.1.8 + Pro 3.1.8. |
| `fluentcrm-automation-sequence-models` | Start and inspect automations and Pro email sequences without writing tables directly — and keep the two apart: `FunnelSequence` (an automation step in `fc_funnel_sequences`) vs FluentCampaign `Sequence` (a Pro email sequence in `fc_campaigns`). Covers `FunnelSubscriber` fields/statuses, starting via `FunnelProcessor::startFunnelSequence()` and resuming via `startFunnelFromSequencePoint()` (not manual row inserts), the `ifAlreadyInFunnel` + unique-key duplicate guard, the `subscribed`/`transactional`-only processing gate and `__force_run_actions`, batch filters, reading progress via ORM, and Pro sequence enroll/unenroll through `Sequence::subscribe()` / `unsubscribe()` with a `SequenceTracker` dedupe check. The canonical `last_sequence_status` is `complete` (not `completed`). Tested FluentCRM 3.1.8 + Pro 3.1.8. |
| `fluentcrm-smartcodes-segments` | Add and parse custom merge tags and build Pro dynamic segments. `FluentCrmApi('extender')->addSmartCode()` (group key, callback `($code, $valueKey, $defaultValue, $subscriber)`, reserved group keys to avoid), the `{{group.key}}` / `##group.key##` syntax with `|` defaults and `||` transformers, parsing via `fluent_crm/parse_campaign_email_text` / `Parser::parse` and the deliberately-deferred unsubscribe/manage URLs, `fluent_crm_funnel_context_smart_codes` for automation-context tags, Pro dynamic segments (`fluentcrm_dynamic_segments` + `fluentcrm_dynamic_segment_{slug}` returning an ORM query model, not an array), and `fluentcrm_contacts_filter_*` advanced-filter providers (mutate the query by reference, allowlist every property/operator). Privacy/escaping checklist throughout. Tested FluentCRM 3.1.8 + Pro 3.1.8. |

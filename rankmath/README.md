# rankmath

Compatibility skills for [Rank Math SEO](https://rankmath.com) (`seo-by-rank-math`). Use when your own plugin or theme must cooperate with Rank Math ‚Äî expose custom content to its metadata, Schema, and sitemap pipelines instead of printing competing SEO output. Grounded against the free plugin source (1.0.273); PRO-only APIs are out of scope unless source-verified.

## Skills

| Skill | Purpose |
|---|---|
| `rankmath-plugin-compatibility` | The base compatibility layer ‚Äî bootstrap safely around `rank_math/loaded` (fires before `plugins_loaded:14` init and `after_setup_theme:2` module load; handle both orders, keep Rank Math optional), make CPTs/taxonomies discoverable (fix `public`/`publicly_queryable` registration first; `rank_math/excluded_post_types` is a list filter despite the name), filter frontend metadata via the narrowest hook (`rank_math/frontend/title`, `/description`, `/robots` ‚Äî associative array, `/canonical`, `rank_math/opengraph/*`), register replacement variables on `rank_math/vars/register_extra_replacements`, feed custom editor fields into content analysis via the `rank_math_content` JS filter (the documented `rank_math_title` JS filter was NOT verifiable in the 1.0.273 build), and headless `getHead` compatibility. Module ID gotcha ‚Äî Schema is `rich-snippet`, not `schema`. |
| `rankmath-schema-integration` | Extend Rank Math's single JSON-LD `@graph` instead of printing a second script ‚Äî `rank_math/json_ld` at priority 100 (entity connector runs at 99), type-specific `rank_math/snippet/rich_snippet_{type}_entity`, breadcrumb and validated-data hooks. Never `array_values()` the graph (later callbacks use associative keys), don't assume the `Product`/`Article` key name (stored schemas key off `meta_id`, `@type` may be an array), stable `@id`s derived from the canonical URL (no `uniqid()`/render-time randomness), reference entities by `['@id' => ...]`, never invent ratings/prices/authors, and don't write `rank_math_schema_*` postmeta directly for runtime integration. |
| `rankmath-sitemap-integration` | Include third-party content in Rank Math XML sitemaps ‚Äî built-in post-type/taxonomy providers first, then the narrowest filter (`rank_math/sitemap/entry`, `xml_post_url`, `exclude_post_type`, index entries), symmetric `get_posts`/`post_count` join+where SQL filters (filtering only one side yields sparse/empty pages or truncation), and custom providers (`handles_type`/`get_index_links`/`get_sitemap_links` contract, one-based `$current_page`, stable ordering tiebreaker, absolute same-site URLs) only for genuinely virtual resources ‚Äî without `implements` on the Rank Math interface in always-loaded files (fatals when Rank Math is inactive). Cache invalidation via normal WP mutations (Cache Watcher), `rank_math/sitemap/invalidate_object_type` for posts, guarded `Cache::invalidate_storage()` (clears ALL sitemap storage ‚Äî debounce during imports) for custom tables. |

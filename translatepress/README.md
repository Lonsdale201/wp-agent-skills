# translatepress

Compatibility skills for the **[TranslatePress](https://translatepress.com)** multilingual plugin — the free core **TranslatePress Multilingual** (`translatepress-multilingual`) plus the paid **TranslatePress Business** add-ons (SEO Pack, Multiple Domains, Navigation Based on Language, Translator Accounts, Browse as User Role, Automatic User Language Detection).

These skills are for making **your own** plugin or theme behave correctly when TranslatePress is active — they are not about extending TranslatePress itself. TranslatePress translates the rendered front-end page (plus gettext and dynamic strings) rather than storing per-language content, so companion code has to render stable, server-rendered output, use WordPress URL/menu/mail APIs instead of hand-built language logic, exclude only the values that must stay literal, and vary caches by language and host. Use these when building or auditing a plugin/theme that has to ship as "TranslatePress-compatible".

## Skills

| Skill | Purpose |
|---|---|
| `translatepress-output-compatibility` | Make plugin/theme output translatable: render real front-end text (not opaque data), keep dynamic JS markup stable, vary HTML caches by language (and host under Different Domain per Language), and exclude only the values that must stay literal via `data-no-translation` / `data-no-auto-translation` / `data-no-dynamic-translation` / `data-no-translation-href` and the `trp_no_translate_selectors` / `trp_no_auto_translate_selectors` / `trp_skip_selectors_from_dynamic_translation` filters. Covers gettext discoverability, nonces/tokens/IDs, AJAX/REST fragments, and the Automatic Language Detection cookie/popup interaction. Tested against core 3.2.1 + Business 1.8.2. |
| `translatepress-url-seo-compatibility` | Keep URLs, routing, slugs, and SEO output correct under TranslatePress and the Business SEO Pack / Multiple Domains add-ons: use WordPress URL APIs and the documented `url_converter` instead of manual `/en/` prefixing, register CPT/taxonomy rewrites deterministically (SEO Pack owns translated slugs), avoid raw `REQUEST_URI` routing, use the `trp_is_rest_api` / `trp_is_admin_link` / `trp_is_form_for_request_uri` / `trp_allow_redirect_to_translated_url` filters for custom endpoints, handle translated WooCommerce permalinks, mark external links with `data-no-translation-href`, and never cache absolute URLs / canonical / sitemap / JSON-LD across languages or mapped domains. Tested against core 3.2.1 + Business 1.8.2. |
| `translatepress-language-ui-navigation` | Build TranslatePress-aware language UI and navigation: prefer built-in switchers, then `trp_custom_language_switcher()` with a `data-no-translation` wrapper (never hand-built switcher URLs); use the current core `[language-include]` / `[language-exclude]` conditional shortcodes (legacy `[trp_language]` is deprecated); render menus through `wp_nav_menu()` / `wp_get_nav_menu_items()` so Navigation Based on Language can hide language-limited items; preserve Translator Accounts (`translate_strings`) admin-bar access; avoid write side effects during Browse as User Role / translation-editor previews; and keep the Automatic User Language Detection popup/hello bar uncovered. Tested against core 3.2.1 + Business 1.8.2. |
| `translatepress-email-notification-compatibility` | Send plugin / WooCommerce / transactional emails in the recipient's language: prefer `wp_mail()` (TranslatePress filters subject/body and switches to the first recipient's preferred language), send separate emails per language, switch with `trp_switch_language()` / `trp_restore_language()` around custom-rendered templates and `trp_translate()` for isolated strings, read durable language from `trp_language` user/order meta for cron/CLI/webhook jobs, preserve WooCommerce order `trp_language` meta (HPOS-safe, no raw SQL), and correct edge cases with `trp_woo_email_language`. Never translate names, addresses, order numbers, SKUs, tokens, or payment IDs. Core-side email translation lives in `translatepress-multilingual`; tested with Business 1.8.2 present. |

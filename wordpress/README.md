# wordpress

Core WordPress skills that apply to any plugin or theme ŌĆö security (incl. file uploads & outbound HTTP), privacy tooling, i18n, REST, the Abilities API, HTML/UTF-8 helpers, query cache.

Use these when reviewing or extending code that touches WordPress core APIs, regardless of which plugin you're building.

## Skills

| Skill | Purpose |
|---|---|
| `wp-security-audit` | Basic security checklist ŌĆö sanitize, escape, nonce, capability, SQL prepare, AJAX nopriv, REST permission, redirects, path traversal. |
| `wp-security-deep` | Deeper security ŌĆö object injection, SSRF, CSRF on GET, mass assignment, file include, mail/zip injection, timing comparison, TOCTOU. |
| `wp-security-secrets` | Secrets handling ŌĆö hardcoded credentials, weak randomness, password storage, cookie flags, log leaks. |
| `wp-i18n-audit` | Translation correctness ŌĆö text-domain consistency, escaped translation calls, placeholder helpers, translator comments. |
| `wp-settings-storage-audit` | Audit how a plugin or classic theme persists settings ŌĆö the storage-primitive decision (option / theme_mod / meta / transient / site option / custom table), stable prefixed keys and documented array shapes, Settings API save invariants, sanitize-as-schema with defaults and tab-merge, deliberate autoload (the WP 6.6+ `auto`/`on`/`off` model, no legacy `'yes'`/`'no'`), REST `show_in_rest` schema matching (array `items` / object `properties`), the Customizer `theme_mod` vs `option` boundary, and deprecated settings groups/handlers. Returns a Correct / Risky / Incorrect verdict with a storage map, findings, fix plan, and migration notes. The audit companion to `wp-admin-settings-api`, `wp-plugin-options-storage`, and `classic-theme-customizer`. |
| `wp-rest-api` | Scaffold and review custom REST endpoints ŌĆö `register_rest_route`, `permission_callback` (the `__return_true` antipattern), `args` schema, response shaping with `WP_REST_Response` / `WP_Error`, REST vs admin-ajax decision. Suggests graduating to `better-route` for non-trivial multi-route projects. |
| `wp-abilities-api` | The WordPress Abilities API (WP 6.9 server-side, WP 7.0 client-side) ŌĆö register machine-readable plugin capabilities with JSON Schema inputs/outputs and a permission callback, automatically exposed over REST and bridgeable to AI agents (via MCP adapter or the WP 7.0 WP AI Client). Covers `wp_register_ability`, the `@wordpress/abilities` / `@wordpress/core-abilities` script modules, `wp_ai_client_prompt()->using_abilities()`, and the REST schema cleanup in 7.0. |
| `wp-connectors-api` | The WordPress Connectors API (WP 7.0+) ŌĆö register external-service connection metadata for the Settings > Connectors screen and WP AI Client. Covers `wp_connectors_init`, `WP_Connector_Registry`, `wp_get_connector` / `wp_get_connectors` / `wp_is_connector_registered`, `api_key` vs `none` auth, env / constant / option key priority, AI provider auto-discovery, and safe metadata override patterns. |
| `wp-ai-client` | The WordPress 7.0 WP AI Client ŌĆö provider-agnostic text / image / speech / video / JSON / ability-powered generation. Covers `wp_ai_client_prompt()`, `WP_AI_Client_Prompt_Builder`, `wp_supports_ai`, `using_model_preference`, `is_supported_*` capability gating, `generate_*` / `generate_*_result` methods, `WP_Error` handling, `using_abilities()`, `WP_AI_Client_Ability_Function_Resolver` allowlist, connector-backed provider configuration, and prompt prevention filters. |
| `wp-presence-api` | **Awareness-only** reference for the experimental Presence API feature plugin (v0.1.x, April 2026 ŌĆö NOT in core, NOT a stable release). Surfaces the durable architectural pattern (dedicated ephemeral table + TTL + Heartbeat) so AI assistants don't invent `postmeta`-based presence implementations. |
| `wp-html-api` | Safe server-side HTML inspection/mutation with `WP_HTML_Tag_Processor` and `WP_HTML_Processor` ŌĆö replaces regex/string hacks for tag attributes, classes, text nodes, token serialization, and `data-*` name mapping. |
| `wp-utf8-text` | UTF-8/text encoding handling for imports, exports, XML/JSON/feed boundaries, logs, and external API data ŌĆö WP 6.9 `wp_is_valid_utf8()`, `wp_scrub_utf8()`, noncharacter checks, and the `seems_utf8()` deprecation. |
| `wp-query-cache` | Query-cache behavior on WP 6.9+ ŌĆö salted cache helpers for direct query cache access, affected groups (`post-queries`, `term-queries`, `user-queries`, etc.), and when to avoid touching core query caches directly. |
| `wp-admin-settings-api` | Build plugin settings pages with the Settings API instead of custom POST handlers ŌĆö `register_setting()`, `add_settings_section()`, `add_settings_field()`, `settings_fields()`, `do_settings_sections()`, `<form action="options.php">`, `sanitize_callback`, single-array storage, tabbed pages, `show_in_rest`, and `option_page_capability_{$option_group}`. |
| `wp-admin-list-table` | Build admin record tables by extending `WP_List_Table` ŌĆö `prepare_items()`, columns, sortable / bulk actions, pagination, row actions, search, views, Screen Options per-page, and the `check_admin_referer( 'bulk-' . $plural )` CSRF guard plugins forget. |
| `wp-admin-form-controls` | Core admin form widgets ŌĆö `wp-color-picker`, `jquery-ui-datepicker` (with the missing CSS), `jquery-ui-autocomplete`, and `wp-pointer`. Covers correct enqueues, change / clear callbacks, `yy-mm-dd` formatting + strict server sanitization, autocomplete `source` shapes, core user/tag suggest, and `dismiss-wp-pointer`. |
| `wp-admin-media-frame` | Open the standard Media Library picker from plugin admin ŌĆö `wp_enqueue_media()` + `wp.media()`, library filters (type / MIME / uploadedTo / author), `multiple` and `'add'` modes, `select` / `open` events, attachment `sizes`, frame caching, pre-selected attachments, and saving IDs (not URLs). |
| `wp-admin-codemirror` | Embed the bundled CodeMirror editor in admin via `wp_enqueue_code_editor()` + `wp.codeEditor.initialize()`. Covers MIME / file mode selection for CSS / JS / JSON / HTML / PHP / SQL / Markdown / YAML, the `false` return when user profile syntax highlighting is disabled, linter handles (`csslint`, `htmlhint`, `htmlhint-kses`, `jsonlint`), and `wp_code_editor_settings`. |
| `wp-admin-drag-and-drop` | Build admin drag-and-drop UI with core's bundled jQuery UI Sortable / Draggable / Droppable, optional `jquery-touch-punch`, `wp-api-fetch`, and `wp-a11y.speak()`. Covers flat reorder, connected lists / kanban columns, palette-to-dropzone clones, hierarchical tree indentation, REST order persistence, and keyboard reorder controls. |
| `wp-admin-postbox-sortable` | Wire up WordPress postboxes on custom admin pages with collapse, drag sorting, Screen Options visibility, and core persistence ŌĆö `add_meta_box()`, `do_meta_boxes()`, the `postbox` script, `postboxes.add_postbox_toggles()`, the `.meta-box-sortables` / `.postbox` / `.hndle` DOM contract, and the two nonces plugins forget (`closedpostboxesnonce`, `meta-box-order-nonce`). |
| `wp-admin-notices` | Render admin notices via the four core hooks (`admin_notices`, `network_admin_notices`, `user_admin_notices`, `all_admin_notices`) and the 6.4+ `wp_admin_notice()` / `wp_get_admin_notice()` helpers. Covers severity classes, `is-dismissible`, per-user persisted dismissal via `user_meta` + REST, screen targeting via `get_current_screen()`, transient-backed flash notices after redirects, and the `wp_admin_notice_args` / `wp_admin_notice_markup` filters. |
| `wp-cli-extending` | Add custom WP-CLI commands to a plugin via `WP_CLI::add_command()`. Class-based commands with PHPDoc-driven synopsis, positional vs `--flag` args, I/O helpers (`success` / `log` / `warning` / `error` / `confirm` / `debug`), `WP_CLI\Utils\format_items()` + `--format=table\|csv\|json\|yaml\|count`, progress bars, `WP_CLI::runcommand()`, lifecycle hooks (`before_wp_load`, `before_invoke:<cmd>`, `after_invoke:<cmd>`), and the `defined( 'WP_CLI' ) && WP_CLI` registration guard. |
| `wp-filesystem-api` | Read, write, copy, delete, chmod files via the `WP_Filesystem` abstraction instead of bare PHP. Covers the bootstrap (`file.php` include ŌåÆ `request_filesystem_credentials()` ŌåÆ `WP_Filesystem()` ŌåÆ `$wp_filesystem->*`), the four transports (direct / ssh2 / ftpext / ftpsockets), `FS_METHOD` / `FS_CHMOD_FILE` / `FS_CHMOD_DIR` constants, the credentials form, and when to use `wp_handle_upload()` / `wp_upload_dir()` instead. |
| `wp-locale-and-dates` | Locale-aware date / time / number handling for plugins. The modern (5.3+) `wp_date()`, `current_datetime()`, `wp_timezone()`, `get_gmt_from_date()` / `get_date_from_gmt()`, `mysql_to_rfc3339()`, `number_format_i18n()` ŌĆö plus the legacy foot-guns to avoid (`current_time('timestamp')` returning offset-summed not-quite-Unix, `date_i18n` quirks, `mysql_to_rfc3339` not actually being RFC3339). |
| `wp-accessibility-audit` | Audit or implement accessibility for plugin admin screens, frontend output, and classic themes against WCAG 2.2 A/AA ŌĆö the ARIA-vs-native decision rule, form field labels / `aria-describedby` / `aria-invalid`, icon-button names, keyboard & focus management, `:focus-visible`, `wp.a11y.speak()` live announcements, color/contrast/zoom/reduced-motion, target sizes, landmarks/headings/tables, media alt text, custom-widget APG patterns, plus a severity guide and an audit workflow. |
| `wp-file-upload-security` | Implement or audit secure file uploads & sideloads ŌĆö `media_handle_upload()` / `wp_handle_upload()` / `wp_check_filetype_and_ext()`, strict MIME + extension allowlists, capability & nonce checks, size limits, attachment cleanup, SVG / archive (ZIP) policy, remote-download cleanup, and private file storage. Use when code handles `$_FILES`, multipart forms, REST uploads, imported remote files, or custom download endpoints. |
| `wp-http-api-client` | Implement or audit outbound HTTP ŌĆö `wp_remote_request()` / `wp_safe_remote_get/post/request()`, bounded timeouts / redirects / response sizes, host allowlists (SSRF), JSON handling, auth-header redaction, retries & idempotency, streaming downloads, and test hooks. Use when a plugin calls an external API, webhook, feed, license or OAuth server, remote file, or fetches a user-supplied URL. |
| `wp-privacy-personal-data` | Wire a plugin into the WordPress privacy tools ŌĆö `wp_add_privacy_policy_content()`, personal-data exporters (`wp_privacy_personal_data_exporters`) and erasers (`wp_privacy_personal_data_erasers`), the paged callback contract, retained-data messages, idempotent erasure, custom tables / meta / remote systems, retention, and multisite scope. Use for plugins storing emails, IPs, user identifiers, form submissions, logs, orders, or other personal data. |

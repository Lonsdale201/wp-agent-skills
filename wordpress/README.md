# wordpress

Core WordPress skills that apply to any plugin or theme â€” security, i18n, REST, the Abilities API, HTML/UTF-8 helpers, query cache.

Use these when reviewing or extending code that touches WordPress core APIs, regardless of which plugin you're building.

## Skills

| Skill | Purpose |
|---|---|
| `wp-security-audit` | Basic security checklist â€” sanitize, escape, nonce, capability, SQL prepare, AJAX nopriv, REST permission, redirects, path traversal. |
| `wp-security-deep` | Deeper security â€” object injection, SSRF, CSRF on GET, mass assignment, file include, mail/zip injection, timing comparison, TOCTOU. |
| `wp-security-secrets` | Secrets handling â€” hardcoded credentials, weak randomness, password storage, cookie flags, log leaks. |
| `wp-i18n-audit` | Translation correctness â€” text-domain consistency, escaped translation calls, placeholder helpers, translator comments. |

| `wp-abilities-api` | The WordPress Abilities API (WP 6.9 server-side, WP 7.0 client-side) â€” register machine-readable plugin capabilities with JSON Schema inputs/outputs and a permission callback, automatically exposed over REST and bridgeable to AI agents (via MCP adapter or the WP 7.0 WP AI Client). | `wp-presence-api` | **Awareness-only** reference for the experimental Presence API feature plugin (v0.1.x, April 2026 â€” NOT in core, NOT a stable release). | `wp-admin-list-table` | Build admin record tables by extending `WP_List_Table` â€” `prepare_items()`, columns, sortable / bulk actions, pagination, row actions, search, views, Screen Options per-page, and the `check_admin_referer( 'bulk-' . $plural )` CSRF guard plugins forget. |

| `wp-admin-drag-and-drop` | Build admin drag-and-drop UI with core's bundled jQuery UI Sortable / Draggable / Droppable, optional `jquery-touch-punch`, `wp-api-fetch`, and `wp-a11y.speak()`.
